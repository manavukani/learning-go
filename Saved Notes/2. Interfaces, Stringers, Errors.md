# Interfaces

An _interface type_ is defined as a set of method signatures.

A value of interface type can hold any value that implements those methods.

Note: if `interface` has multiple methods, **any type implementing an interface must have ALL methods** defined in that interface's signature. You cannot omit any methods. Else, throws compile-time errors

```go
type Runner interface {
    Run()
}

type Robot struct{}
func (r *Robot) Run() {}

```

**For CALLS:** Go automatically takes addresses or dereferences.

```go
p := Robot{}
p.Run()        // works (auto address)
```

**For INTERFACES:** Go does **NOT** do any auto-conversion because these are compile-time rules.

``` go
var r Runner 
r = Robot{}    // DOES NOT WORK (method set mismatch)
```

Interfaces require _actual_ method presence — no magic rules.

### Another example:
`Vertex` (the value type) doesn't implement `Abser` because the `Abs` method is defined only on `*Vertex` (the pointer type).

```go
package main

import (
	"fmt"
	"math"
)

type Abser interface {
	Abs() float64
}

func main() {
	var a Abser
	f := MyFloat(-math.Sqrt2)
	v := Vertex{3, 4}

	a = f  // a MyFloat implements Abser
	a = &v // a *Vertex implements Abser

	// In the following line, v is a Vertex (not *Vertex)
	// and does NOT implement Abser.  --- gives error
	// a = v

	fmt.Println(a.Abs())
}

type MyFloat float64

func (f MyFloat) Abs() float64 {
	if f < 0 {
		return float64(-f)
	}
	return float64(f)
}

type Vertex struct {
	X, Y float64
}

func (v Vertex) Abs() float64 {
	return math.Sqrt(v.X*v.X + v.Y*v.Y)
}
```

### Interface values
Under the hood, interface values can be thought of as a **tuple of a value and a concrete type**:

```go
(value, type)
```

An interface value holds a value of a specific underlying concrete type.

Calling a method on an interface value executes the method of the same name on its underlying type.

```go
package main

import (
	"fmt"
	"math"
)

type I interface {
	M()
}

type T struct {
	S string
}

func (t *T) M() {
	fmt.Println(t.S)
}

type F float64

func (f F) M() {
	fmt.Println(f)
}

func main() {
	var i I

	i = &T{"Hello"}
	describe(i) // op =  (&{Hello}, *main.T)
	i.M() // op = Hello

	i = F(math.Pi)
	describe(i) // op = (3.141592653589793, main.F)
	i.M() // op = 3.141592653589793
}

func describe(i I) {
	fmt.Printf("(%v, %T)\n", i, i)
}
```

### Nil Interface

>interface can hold a `nil` concrete value (like `t`), but the interface itself is **not** `nil` unless the concrete value ***and*** the type are both `nil`.

- **Nil Pointer Receiver**: 
	- When a method has a pointer receiver (like `func (t *T) M()`), 
		- Go: allows the method to be called even with a `nil` pointer. 
		- Other languages: *null pointer exception*
	- Good practice to check for `nil` inside methods like this to avoid runtime errors or panic

```go
package main

import "fmt"

type I interface {
	M()
}

type T struct {
	S string
}

func (t *T) M() {
	if t == nil { // <================================ IMP
		fmt.Println("<nil>")
		return
	}
	fmt.Println(t.S)
}

func main() {
	var i I // initialize interface

	var t *T // initialize type
	
	i = t // can be directly init as: var i I = &T{}
	describe(i)         // (<nil>, *main.T)
	i.M()               // <nil>

	i = &T{"hello"}
	describe(i)         // (&{hello}, *main.T)
	i.M()               // hello
}

func describe(i I) {
	fmt.Printf("(%v, %T)\n", i, i)
}
```


A nil interface value holds neither value nor concrete type.
```go
// calling uninitialized interface (i) method (M) with no type assigned (var t *T)
func main() {
	var i I
	describe(i)
	i.M()
}

```

### Empty Interface
The interface type that specifies ***zero methods*** is known as the _empty interface_.

```go
interface{}
```

An empty interface may **_hold values of any type_**. (Every type implements at least zero methods.)

Empty interfaces are used by code that handles values of unknown type.

```go
// PROBLEM: cannot use ONLY array/slice --- inputs are different types (string vs uint)
// SOLUTION: use slice for prompting, interface{} allows different types for scan
func getUserInput() (string, string, string, uint) {
	var firstName, lastName, email string
	var userTickets uint

	prompts := []struct {
		message string
		target  interface{}
	}{
		{"Enter Your First Name: ", &firstName},
		{"Enter Your Last Name: ", &lastName},
		{"Enter Your Email: ", &email},
		{"Enter number of tickets: ", &userTickets},
	}

	for _, prompt := range prompts {
		fmt.Print(prompt.message)
		fmt.Scanln(prompt.target)
	}

	return firstName, lastName, email, userTickets
}
```

### Type Assertion in Interface

A _type assertion_ provides access to an interface value's underlying concrete value.

If `i` does not hold a `T`, the statement will trigger a ***panic***.

```go
t := i.(T)
t, ok := i.(T) // optional boolean value for success
```

Unless, we use the optional `ok`
- If `i` holds a `T`, then `t` will be the underlying value and `ok` will be true.
- If not, `ok` will be false and `t` will be the zero value of type `T`, and ***no panic occurs***.

### Type Switches for Interface

A _type switch_ is a construct that permits several type assertions in series.
- type switch is like regular switch statement, but cases in type switch specify types (not values)
- declaration in type switch has same syntax as type assertion `i.(T)`, but specific type `T` is replaced with the keyword `type`

```go
package main

import "fmt"

func do(i interface{}) {
	switch v := i.(type) {
	case int:
		fmt.Printf("Twice %v is %v\n", v, v*2)
	case string:
		fmt.Printf("%q is %v bytes long\n", v, len(v))
	default:
		fmt.Printf("I don't know about type %T!\n", v)
	}
}

func main() {
	do(21) // Twice 21 is 42
	do("hello") // "hello" is 5 bytes long
	do(true) // I don't know about type bool!
}
```

# Stringers
One of the most ubiquitous interfaces is [`Stringer`](https://go.dev/pkg/fmt/#Stringer) defined by the [`fmt`](https://go.dev/pkg/fmt/) package.

```go
type Stringer interface {
    String() string
}
```

A `Stringer` is a type that can describe itself as a string. The `fmt` package (and many others) look for this interface to print values.

```go
package main

import "fmt"

type IPAddr [4]byte

// TODO: Add a "String() string" method to IPAddr.
func (ip IPAddr) String() string {
	return fmt.Sprintf("%d.%d.%d.%d", ip[0], ip[1], ip[2], ip[3])
}
// custom styling "just for IPAddr"

func main() {
	hosts := map[string]IPAddr{
		"loopback":  {127, 0, 0, 1},
		"googleDNS": {8, 8, 8, 8},
	}
	for name, ip := range hosts {
		fmt.Printf("%v: %v\n", name, ip)
	}
}

```

# Errors

The `error` type is a built-in interface,  `fmt` package looks for `error` interface when printing

```go
type error interface {
    Error() string
}
```

Functions often return an `error` value, and calling code should handle errors by testing whether the error equals `nil`.

```go
package main

import (
	"fmt"
	"time"
)

type MyError struct {
	When time.Time
	What string
}

func (e *MyError) Error() string {
	return fmt.Sprintf("at %v, %s",
		e.When, e.What)
}

func run() error {
	return &MyError{
		time.Now(),
		"it didn't work",
	}
}

func main() {
	if err := run(); err != nil {
		fmt.Println(err)
	}
}
```

>A nil `error` denotes success; a non-nil `error` denotes failure.

# ==EXAMPLE - IMP==

`Sqrt` should return a non-nil error value when given a negative number, as it doesn't support complex numbers.

Create a new type: `type ErrNegativeSqrt float64` and make it an `error` by giving it a `func (e ErrNegativeSqrt) Error() string` method such that:
`ErrNegativeSqrt(-2).Error()` returns `"cannot Sqrt negative number: -2"`.

**Note:** 
- A call to `fmt.Sprint(e)` inside the `Error` method will send the program into an infinite loop.
- Inside `Error()`:
	- `fmt.Sprint(e)` tries to print `e`
	- printing `e` calls `Error()` again
	- which calls `fmt.Sprint(e)` again
	- and so on forever
- You can avoid this by converting `e` first: `fmt.Sprint(float64(e))`.

```go
package main

import (
	"fmt"
)

type ErrNegativeSqrt float64


// ErrNegativeSqrt implements error interface
func (e ErrNegativeSqrt) Error() string {
	return fmt.Sprintf("cannot Sqrt negative number: %f", float64(e)) // !inf loop
}

func Sqrt(x float64) (float64, error) {
	if x < 0 {
		return 0, ErrNegativeSqrt(x) // implements the error
	}

	z := float64(1)
	for i := 0; i < 10; i++ {
		z -= (z*z - x) / (2*z)
	}
	return z, nil
}

func main() {
	fmt.Println(Sqrt(2))
	fmt.Println(Sqrt(-2))
}
```

> You didn’t _throw_ an error. You _returned_ a value that **happens to implement an interface**.
